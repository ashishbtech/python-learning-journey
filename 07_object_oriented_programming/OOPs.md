# 🏷 Object-Oriented Programming (OOP) – Revision Guide

This document provides a **concise revision of OOP concepts in Python**, including terminology, best practices, and learning roadmap. Ideal for quick review.

---

## 1️⃣ Core Concepts

| Concept | Description |
|---------|-------------|
| **Class** | Blueprint for creating objects; defines attributes and behaviors. |
| **Object** | Instance of a class; holds actual data. |
| **Attributes** | Variables associated with an object. |
| **Methods** | Functions inside a class that manipulate its data. |
| **Constructor (`__init__`)** | Special method that initializes object attributes. |
| **Inheritance** | Mechanism to allow a class to acquire attributes/methods from another class. |
| **Polymorphism** | Objects of different classes responding differently to the same method. |
| **Encapsulation** | Bundling data and methods together; restricts direct access when needed. |

---

## 2️⃣ Principles of OOP

1. **Abstraction** – Show essential features, hide implementation details.  
2. **Encapsulation** – Keep data safe; interact via methods.  
3. **Inheritance** – Reuse code from existing classes.  
4. **Polymorphism** – Uniform interface with different behaviors.
